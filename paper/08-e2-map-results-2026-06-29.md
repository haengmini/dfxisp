# E2 실측 mAP — pseudo-RAW COCO + ExDark

파이프라인: `raw .bin(RGGB16) → demosaic → variant ISP → YOLO → mAP(ultralytics val)`.
도구: `tools/eval_map_coco.py`(+`--remap`), `tools/eval_map_dark.py`. 데이터: `DFXISP/dataset/{COCO_5000_raw,ExDark_5000_raw}/val`.
(ExDark 라벨은 이미 COCO-80 id로 저장돼 있어 remap 불필요.)

## 결과 (yolov8n, mAP@[.5:.95])
| variant | COCO 전체 n=347 | COCO dark-40 | COCO dark-60 | **ExDark real-lowlight n=260** |
|---|---|---|---|---|
| static   | 0.2984 | 0.3565 | 0.2966 | 0.1492 |
| reg_only | 0.2952 | 0.3619 | 0.2912 | **0.1585** |
| dfx_bin  | 0.2455 | 0.3693 | 0.2305 | 0.1420 |
| dfx_fp   | 0.2518 | 0.3179 | 0.2553 | **0.0879** |

## 견고한 결론
1. **적응형 스위칭 필요(Contribution 1/3 지지):** 정상조도 위주 COCO 전체에서 저조도 처리는 이득 없음(static/reg ≥ DFX). 저조도 RM은 장면 적응적으로만 켜야 함.
2. **핵심 가설(Contribution 2) 반증:** **real low-light(ExDark)에서 DFX-FP가 압도적 최저(0.088), reg_only(0.159)가 최고.** "feature-preserving DFX-FP가 저조도 detection mAP에서 우월"은 예비 실측이 **분명히 지지하지 않음.** FP의 detail add-back이 어두운 영역의 demosaic/양자화 아티팩트를 증폭해 detection을 악화.
3. **DFX 구조적 RM의 mAP 이점 불명확:** ExDark에서 DFX-Bin(0.142)·DFX-FP(0.088) 모두 단순 register gain(0.159)을 못 이김 → 저조도 mAP만 보면 **register-only가 강력한 baseline**.

## detector 교차검증 (견고성 확인)
ExDark(n=260)를 yolov8s로 재평가 → **순서 동일**: reg_only(0.190) > static(0.171) > dfx_bin(0.151) > dfx_fp(0.102).
즉 **"reg_only 최고 / DFX-FP 최저" 결론은 detector 크기에 무관하게 견고.**

| variant | ExDark yolov8n | ExDark yolov8s |
|---|---|---|
| static | 0.1492 | 0.1714 |
| reg_only | **0.1585** | **0.1902** |
| dfx_bin | 0.1420 | 0.1511 |
| dfx_fp | 0.0879 | 0.1018 |

## 결론 불가/주의
- COCO dark-subset 순서는 subset마다 뒤집힘(dark40 Bin>FP, dark60 FP>Bin) → 소표본 노이즈. 단, **ExDark(real, n=260)에선 FP 최저가 두 detector 모두 큰 격차로 안정적.**
- caveat: pseudo-RAW noise 비활성(ExDark 원본은 실제 저조도라 어두움 특성은 보존). demosaic RGGB(데이터셋) vs GRBG(C-sim golden). 향후 real-RAW+노이즈로 추가 확인.

## 함의 (논문/설계)
- **DFX-FP를 "저조도 mAP 향상 RM"으로 주장하기 어려움.** 재포지셔닝 필요:
  (a) 저조도 main RM을 **denoise 지향**으로(현 FP의 detail boost 제거/역전), 또는
  (b) DFX 가치를 mAP가 아니라 **자원/전력 절감**(register-only 대비 normal 모드 fabric 축소)으로 재정의, mAP는 register gain으로 확보.
- **register-only가 저조도 mAP에서 최강** → 논문 서사를 "register-fast가 대부분을 해결하고 DFX는 자원/전력에서만 정당"으로 좁히는 것이 데이터와 정합.
- real RAW + 노이즈 + 큰 detector로 확정 권장.

## 부록 — 방향 A 후속: DFX-Bin guardrail 대응 (2026-06-29)
nearest 복제 업샘플 → **integer bilinear(9/3/3/1)** 로 개선한 DFX-Bin 재측정:

| regime | DFX-Bin nearest | **DFX-Bin bilinear** | register-only | guardrail(≤1.0pt) |
|---|---|---|---|---|
| ExDark (저조도, RM 동작 regime) | 0.1420 | **0.1576** | 0.1585 | **PASS (0.09pt)** |
| COCO 전체 (RM 미동작 regime) | 0.2455 | 0.2839 | 0.2952 | 1.13pt(초과; 스케줄러가 binning 미사용) |

**결론:** bilinear DFX-Bin은 **실제 동작 regime(저조도)에서 mAP guardrail 통과** → 방향 A의 DFX 1순위 RM으로 확정. C-sim(정수 bilinear)도 bit-exact 반영(2816px 0 mismatch). 정상조도의 1.13pt 초과는 RM이 켜지지 않는 영역이라 무의미(필요 시 경량 unsharp로 추가 보정 가능).
